# R&D Knowledge Graph & Search Engine

## Архитектура проекта

*   **`api/`** — Бэкенд-сервер приложения:
    *   `main.py` — Точка входа в FastAPI (настройка жизненного цикла приложения и подключение роутера).
    *   `routes.py` — Эндпоинт `/ask` для обработки поисковых запросов пользователей.
    *   `dependencies.py` — Управление асинхронными сессиями базы данных.
*   **`core/`** — Алгоритмическое ядро и искусственный интеллект:
    *   `schemas.py` — Структура данных Pydantic (схемы фактов, параметров и валидация типов).
    *   `prompts.py` — Шаблоны системных инструкций для разметки, генерации SQL и синтеза ответов.
    *   `llm_client.py` — Асинхронный клиент для отправки сетевых запросов к LLM API.
    *   `extractor.py` — Оркестратор преобразования сырого неструктурированного текста в JSON-объекты.
    *   `retriever.py` — Поисковый движок (декомпозиция запросов, трансляция в SQL и резервный поиск).
    *   `logger.py` — Настройка логирования событий с автоматическим созданием файлов в `logs/`.
*   **`parsers/`** — Инструменты предобработки текстов:
    *   `document_parser.py` — Чтение PDF-файлов и извлечение сырого текстового слоя.
    *   `text_chunker.py` — Нарезка длинных текстов на перекрывающиеся фрагменты (чанки).
*   **`db/`** — Слой хранения данных:
    *   `database.py` — Конфигурация асинхронного подключения к СУБД SQLite.
    *   `models.py` — SQLAlchemy-модели таблиц базы данных (`facts` и `parameters`).
    *   `crud.py` — Логика записи извлеченных ИИ фактов в реляционную структуру БД.
*   **`scripts/`** — Служебные скрипты автоматизации:
    *   `ingest_data.py` — Конвейер пакетной обработки PDF-документов и наполнения БД.
    *   `clean_db.py` — Скрипт очистки и нормализации географических дубликатов в базе данных.
    *   `test_db.py` — Быстрый отладочный вывод первых записей из таблицы фактов.
*   **`frontend/`** — Пользовательский интерфейс:
    *   `app.py` — Чат-интерфейс ассистента для общения с базой знаний.
    *   `pages/dashboard.py` — Интерактивная аналитическая панель со статистикой данных.
*   **`.streamlit/`** — Конфигурация интерфейса:
    *   `config.toml` — Отключение сбора статистики и оптимизация отображения Streamlit.
    *   `credentials.toml` — Обход приветственного экрана Streamlit при первом запуске.
