# R&D Knowledge Graph & Search Engine

## Архитектура

*   **`core/`** — Бизнес-логика и ИИ:
    *   `schemas.py` — Схемы данных Pydantic (структура фактов, параметров, единиц измерения).
    *   `prompts.py` — Системные инструкции для извлечения данных и генерации SQL.
    *   `llm_client.py` — Асинхронный клиент для работы с API YandexGPT.
    *   `extractor.py` — Модуль структурирования сырого текста в JSON.
    *   `retriever.py` — Поисковый движок (трансляция вопроса в SQL-запрос и сборка ответа).
*   **`parsers/`** — Чтение документов:
    *   `document_parser.py` — Извлечение сырого текста из PDF-файлов (PyMuPDF).
    *   `text_chunker.py` — Нарезка длинных текстов на перекрывающиеся части.
*   **`db/`** — Хранение данных:
    *   `database.py` — Настройка асинхронного подключения к SQLite.
    *   `models.py` — Таблицы базы данных SQLAlchemy (`facts`, `parameters`).
    *   `crud.py` — Запись извлеченных фактов в базу данных.
*   **`api/`** — Бэкенд:
    *   `main.py` — Точка входа в FastAPI.
    *   `routes.py` — Эндпоинт `/ask` для обработки поисковых запросов.
    *   `dependencies.py` — Управление сессиями базы данных.
*   **`frontend/`** — Интерфейс:
    *   `app.py` — Чат-интерфейс для пользователя.
    *   `pages/dashboard.py` — Аналитический дашборд со статистикой базы данных.
*   **`scripts/`** — Служебные скрипты:
    *   `ingest_data.py` — Скрипт первоначального наполнения БД из папки документов.
